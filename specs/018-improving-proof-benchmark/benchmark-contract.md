# Spec 018 - Improving Proof Benchmark Contract

**Status:** preregistration draft
**Goal:** graduate the living-agent verdict from APPLIED to IMPROVING without weakening thresholds.

## Cases

1. **DESIGN:OS promotion page** - product positioning, proof narrative, generated examples,
   learning story, conversion, and complete-page investment.
2. **Nutrition native mobile interface** - onboarding, daily overview, capture and AI analysis,
   planning/discovery, connected states, gestures, accessibility, and native platform behavior.
3. **Architecture premium-service page** - site, light, material, process, built proof, and inquiry.

The Nutrition case is not a responsive website in a phone frame. Production motion uses native
platform transitions, gestures, and haptics. A GSAP web prototype may communicate the motion
specification but cannot count as native implementation evidence.

## Motion boundary

DESIGN:OS promotion and Architecture each require one story-bearing GSAP signature interaction.
Supporting choreography must share its duration, easing, and spatial grammar. Every scene records:

```text
trigger -> changed property -> user meaning -> reduced-motion equivalent
```

Evaluation covers narrative contribution, spatial continuity, response and interruption,
state completeness, mobile ergonomics, reduced-motion quality, frame stability, INP, CLS,
cleanup, and console errors. Decorative complexity receives no credit.

## Controlled protocol

- Same product facts, assets, model, environment, time budget, and repair budget.
- Control excludes recalled project learning; treatment includes DESIGN:OS recall and orchestration.
- Anonymous randomized evaluation at required desktop/mobile/native device sizes.
- Contradiction cases inject an irrelevant or conflicting lesson; treatment must reject it.
- Isolation cases verify that private project rules do not leak across projects.
- Preserve all attempts and machine-gate envelopes.

## Graduation threshold

- Treatment wins all three cases.
- Mean blind curator delta is at least +10.
- Repair rounds decrease at least 25 percent.
- Repeated user corrections decrease at least 70 percent.
- Zero critical regressions.
- Contradiction and isolation safeguards pass.

Failure produces a learning record naming helpful, harmful, and irrelevant lessons. It never
lowers the threshold or promotes a single-case preference into universal knowledge.
