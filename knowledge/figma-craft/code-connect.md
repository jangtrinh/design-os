# Code Connect — mapping Figma components to code (and what a Free seat can still use)

The bridge between a Figma component and the real code component it represents. When a file
is Code-Connected, Figma's Dev Mode shows *your* component's code (props, import path,
example usage) instead of generic autogenerated CSS — codegen emits the thing engineers
actually paste. This reference exists so ease-design knows **which half of Code Connect is
seat-gated and which half works on any seat**.

## Two halves — authoring vs discovery

**Authoring (`.figma.ts` / `.figma.tsx` templates) needs a PAID Org/Enterprise seat.**
Publishing a Code Connect mapping — writing a `*.figma.tsx` template that binds a code
component to a Figma node key and running `figma connect publish` — requires **Organization
or Enterprise** (Dev Mode is a paid capability). On Figma **Free/Starter this is a hard gap**:
you cannot publish Code Connect mappings. Do not plan a workflow that depends on authoring or
publishing them for a free-seat user (consistent with `figma-agent-hand.md` → Bridge
selection: the free path has no Dev-Mode/MCP surface).

**Discovery — resolving component KEYS by reading `*.figma.tsx` next to component source — is
usable on ANY seat.** A Code Connect template file is just source in the repo. If a codebase
already ships `Button.figma.tsx` beside `Button.tsx`, that template hardcodes the Figma
component **key** (and the node URL) it maps to. Reading those files is plain file I/O — no
paid seat, no API entitlement:

- **Find them:** `*.figma.tsx` / `*.figma.ts` files colocated with component source.
- **Extract:** the `figma.connect(Component, '<figma-url-or-key>', { … })` call — the URL /
  node id / component **key** is the stable identifier you can resolve against a scan
  (`scan-design-system`) or `importComponentByKeyAsync` (paid) / instantiate-by-id (free).
- **Also mineable:** the `props` mapping (`figma.string(...)`, `figma.boolean(...)`,
  `figma.instance(...)`, `figma.enum(...)`) tells you how the design's component properties
  map to code props — useful for design→code and code→design fidelity even without publishing.

So: **treat Code Connect as read-mostly on a Free seat** — harvest the key/prop mappings that
already exist in a repo to resolve components precisely (better than name-matching alone),
but never require authoring/publishing a mapping unless the seat is Org/Enterprise.
