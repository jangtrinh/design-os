# Design System: demo  (generation 1, persona liquid-glass / material-surface)

Intent: landing page for a new gym

> ENFORCEMENT: This is the active design system. Any new design **MUST** consume
> only the semantic tokens below and only the registered components below. Adding
> a new component requires `ui registry register`. Adding/changing a token
> requires `ui ds change-token`.

## Naming rules

- Use semantic tokens, never primitives, in markup.
- Component names follow `Category/Variant` PascalCase.
- One canonical name per concept.

## Anti-patterns (from persona)

- opaque backgrounds
- hard shadows
- sharp corners
- flat surfaces
- heavy borders
